# Python para Análise de Dados: Guia de Estudos Assistido por IA com NotebookLM

## 📖 Sobre o Projeto

Este projeto foi desenvolvido como parte do desafio da DIO com o objetivo de explorar o NotebookLM como ferramenta de aprendizagem ativa apoiada por Inteligência Artificial.

O tema escolhido foi **Python para Análise de Dados**, abrangendo conceitos fundamentais da linguagem Python, manipulação de dados com Pandas e NumPy, fundamentos de Engenharia de Dados, Apache Spark, Databricks, Sistemas Distribuídos, Segurança da Informação e LGPD.

Além da construção de conhecimento técnico, o projeto busca demonstrar como a IA pode auxiliar na organização, síntese e revisão de conteúdos complexos.

---

## 🎯 Objetivos de Estudo

* Aprender os fundamentos da linguagem Python.
* Compreender Programação Orientada a Objetos.
* Utilizar Pandas e NumPy para manipulação de dados.
* Conhecer conceitos de Engenharia de Dados.
* Entender o funcionamento do Apache Spark.
* Explorar o ecossistema Databricks.
* Estudar conceitos básicos de Sistemas Distribuídos.
* Conhecer fundamentos de Segurança da Informação.
* Entender a importância da LGPD no tratamento de dados.

## 🤖 Engenharia de Prompts

Durante o processo de estudo, foram realizados diversos testes de prompts para avaliar a capacidade do NotebookLM em sintetizar conteúdos e auxiliar na aprendizagem.

### Prompt 1

**Pergunta:**

> Como utilizar listas e dicionários em Python para organizar dados?

**Resultado:**

O NotebookLM apresentou exemplos práticos de criação, manipulação e iteração de listas e dicionários.

**Aprendizado:**

Prompts diretos geram respostas mais objetivas e fáceis de revisar.

---

### Prompt 2

**Pergunta:**

> Ensine manipulação e raspagem de dados para um estudante iniciante.

**Resultado:**

Foram apresentados conceitos de coleta de dados, bibliotecas Python e boas práticas para obtenção de informações.

**Aprendizado:**

Especificar o nível de conhecimento melhora significativamente a qualidade da resposta.

---

### Prompt 3

**Pergunta:**

> Explique a relação entre Apache Spark e Databricks.

**Resultado:**

Foi possível compreender como o Databricks utiliza o Apache Spark como mecanismo de processamento distribuído e oferece recursos adicionais para engenharia de dados.

**Aprendizado:**

Prompts contextualizados geram respostas mais próximas da realidade profissional.

## ⚠️ Cicatrizes e Dificuldades Encontradas

Durante os experimentos foram identificados alguns desafios:

* Respostas excessivamente longas em perguntas genéricas.
* Conceitos avançados explicados assumindo conhecimento prévio.
* Necessidade de dividir assuntos complexos em perguntas menores.

### Estratégias Utilizadas

* Solicitar respostas em tópicos.
* Pedir exemplos práticos.
* Informar explicitamente o nível de conhecimento desejado.
* Solicitar comparações em formato de tabela.

## 📖 Glossário

### DataFrame

Estrutura tabular utilizada para armazenamento e análise de dados.

### Series

Estrutura unidimensional do Pandas.

### Spark

Framework distribuído para processamento de grandes volumes de dados.

### Databricks

Plataforma baseada em Apache Spark para engenharia e análise de dados.

### Cluster

Conjunto de máquinas trabalhando em conjunto para processamento distribuído.

### ETL

Processo de Extração, Transformação e Carga de dados.

### Lakehouse

Arquitetura que combina características de Data Lakes e Data Warehouses.

### LGPD

Lei Geral de Proteção de Dados, responsável por regulamentar o tratamento de dados pessoais no Brasil.
